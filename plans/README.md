## Findings considered and rejected

Recorded so nobody re-audits them:

- **Path traversal via the `[slug]` routes** — not exploitable. Slugs reach the filesystem only after
  directory enumeration (`readContentBySlug` matches against `readContentIndex`'s listing), and
  `preview-frame` regex-validates before its dynamic import. Defense-in-depth validation is still
  worth adding when convenient, but it is not a vulnerability.
- **XSS via `dangerouslySetInnerHTML` in `content-feed.tsx`** — not a sink. The script body is a
  static literal; the slug is passed through a React-escaped `data-` attribute and read with
  `getAttribute`.
- **"The registry 404s in production"** — false. Verified against the live site:
  `https://uicapsule.com/r/card-stack.json` returns 200 with correct files and deps. Next traces the
  content tree fine.
- **Duplication across the 30 content packages** — looked for hard; it isn't there. One hand-rolled
  `cn` in 17k LOC, one seeded RNG. Content packages are _supposed_ to be standalone (the registry
  depends on it); extracting a shared primitive would break that contract. Do not "DRY" the content
  directory.
- **`packages/ui/src/components/code-block.tsx` is 499 lines** — big but cohesive, one concern, one
  consumer. Leave it. (`layout.tsx` is the file that actually needs splitting — plan 011.)
- **Turbo pipeline tuning** — `turbo.json` is fine at this repo's size. The real CI win is caching in
  the (missing) PR workflow, which plan 003 adds.
- **`shadcn` as a runtime dep of `packages/ui`** — correct as-is. `globals.css` does
  `@import "shadcn/tailwind.css"`, resolved at build time. Moving it to devDependencies breaks the
  CSS build.
- **Deleting auth/tRPC** — considered (one procedure, zero callers) and **rejected by the maintainer**:
  it is being kept for a future feature. Plan 008 makes it correct instead.
